Help Page




"COMPOSITE SCORE"

Indicates the score of the particular instance (from the deposition) compared to the dictionary for the particular candidate.

The scoring for the ligand matching system provides a tuple of five values (A/B/C/D/E). In all cases they are based on a matching of atoms in the instance against the reference dictionary and calculating statistics.

A = heavy atom match (%)

B = match of chiral centers present, independent of handness (%)

C = match of the handness of the chiral centers (%)

D = match of aromatic flags

E = match of bond order

"MATCH STATUS" -- Possible values as defined below

Ligand Match {chem component ID}: exact match with {chem component ID} for all atom types, number of atoms, connectivity, chirality and 3 letter code

No Match: treated as a new ligand.

"Save (and come back later)" button

If you wish to stop and continue ligand validation at a later point, this button will preserve any work you have saved for individual ligands so that you may resume work at another time.

"Finish (all issues are addressed)" button

Once all the mismatches are addressed, click "Finish" to complete the ligand validation.

"Inspect Selected Ligands" button

Press this button to examine in more detail any of the ligands "selected for inspection" in the summary table.

"Back to Summary View" button

Press this button to return to higher level summary view, which provides inventory of the ligands found in your deposition and their match status.

"Save"/"Edit" button for current ligand

Button toggles between "Save" and "Edit" to allow user to alternatively:

"Save" and lock data entered for the ligand in current view.

or

"Edit" to unlock the data entered and make additional changes for the ligand in current view.

"Work With All Instances of" section

This section allows the annotator to work simultaneously on all instances of a given chemical component group (based on ID). Any actions (assignment, comparison, etc.) taken in this section will therefore pertain to all instances in that group.

"TOP HIT"

For the given ligand instance, signifies the best matching candidate from the ligand dictionary.

"ASSIGNMENT STATUS"

Indicates the current assignment of the given ligand instance.

"COMPARE" boxes

Selecting the comparison box will open up a "Comparison Panel" section with a display of the individual instance so that they may be compared with each other.

"ASSIGNMENT" table

Use this table to assign all instances of a particular ligand to a particular code.

"Force Assign As:" box

In this section, will assign a particular ligand code that may not be present in the list (usually due to missing atoms) to all instances of the group.

"Show rerun Search Form" button

Pressing this button will open a box in which one may change the default link radius (bond between ???) and bond radius (distance between ??? and ???). The numbers are a value added to the default built in values. Pressing the "submit" button will rerun the search on this particular instance.

"Assign"/"Un-Assign" button

Will assign/deassign all instances as the same ligand ID.

"Comparison Panel"

In the "Work with All Instances" section, this panel will allow comparison of individual author instances with each other. In the "Instance:" specific section, the panel allows comparison of individual instances with the dictionary candidates. The color of the column header indicates either the model (green) or dictionary (cyan).

"2D" button

Pressing this button will display a 2D representation of the instances and dictionary.

"3D" button

Pressing this button will open up JMOL panels of the instance/dictionary.

"Atom Map"

Pressing this button will open a panel matching equivalent atoms in the entry. If the 3D representation is open, then pressing a particular atom will zoom in on the specific atom.

"Instance: [instance ID]" section

This section will allow assignment of an individual instance. The top table displays the top candidates that are closest matches to the instance.

"ASSIGN AS" button

This radio button will allow assignment of the ligand. Selecting a particular candidate id and pressing the "Assign" button will make the assignment.

"COMPARE" buttons

Selecting the comparison box will add dictionary items to the "Comparison Panel" section which allows comparison of author's ligand instance with dictionary candidates.

"Force Assign As:" box

In this section, will assign a particular ligand code that may not be present in the list (usually due to missing atoms) to just this ligand instance.

"Assign"/"Un-Assign" button

Will assign/deassign just this ligand instance the given ligand ID.

"Edit / Create New Ligand"

This will start the "Chemical Component Editor Module" After creating a new ligand, you need to use the "Show Rerun Search Form"

"Save"/"Edit" button for capture of research data

Button toggles between "Save" and "Edit" to allow user to alternatively:

"Save" and lock data entered for the research data in current view.

or

"Edit" to unlock the data entered and make additional changes for the research data in current view.

Provide the SMILES or InChi string of the ligand used in the binding assay. If the binding assay was performed on a compound close but not identical to the modeled ligand in the structure being deposited, the descriptor should pertain to the subject of the assay.

Provide numeric value only, which may include up to one decimal place. e.g. 6.5

Please provide numeric value only, in Kelvin. e.g. 293

Provide the type of value that was measured by the binding assay, e.g. Kd, IC50.

Provide the value of the measurement in nM

Provide details of the binding assay that is relevant to the deposited model. The ligand 3-letter code is provided as default. E.g. if there are two instances of the ligand in the model, but only one site is biologically relevant to the binding assay, one should provide the chain ID and residue number of the ligand at the relevant site, such as LIG A 500

If applicable, please provide experimental binding data for the ligand of focus. Please provide the sequence of the binding target if it is different from the sequence that was crystallized. In addition, please provide a descriptor for the binding ligand, especially for those ligands whose bound form in the crystal differs in any way from the form used in the binding assay, e.g., enantiomer, racemic mixture, or reagent prior to reaction.

"Close" button

Press this button to indicate that you would like to close this window. Any changes made in this window are saved within the session and seen in other windows.
